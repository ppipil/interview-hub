# Interview Hub 周报

时间范围：`2026-04-21` 至 `2026-04-27`

## 本周概览

本周的重点不是继续堆新页面，而是把 MVP 主链路从“能跑”往“可维护、可运营、可定位问题”推进了一步。现在项目已经不只是一个单次演示 Demo，而是具备了面试官配置、题库维护、问卷录入、数据库出题和反馈复盘的基本闭环。

## 本周完成

### 1. 主流程和路由收口

- 前端首页、设置页、面试页、反馈页已绑定真实地址：
  - `/`
  - `/setup`
  - `/interview`
  - `/feedback`
- Zustand 阶段切换已和浏览器地址同步，解决多个页面共用同一 URL 的问题

### 2. 面试官体系升级

- 明确拆分两类面试官：
  - 系统面试官：`doubao`
  - AI 分身面试官：`secondme_legacy / secondme_visitor`
- 隐藏管理台 `/admin/interviewers` 已支持：
  - 面试官 CRUD
  - skill / prompt / API key 维护
  - 面试官专属题库维护
  - 通用题库按岗位维护

### 3. 面试官问卷可用

- 新增隐藏问卷页 `/interviewer-questionnaire`
- 支持录入：
  - 面试官姓名
  - 稳定用户名
  - 公司风格、岗位、面试风格、考察点、流程模板
  - SecondMe API Key
  - 专属题库文本
- 同一用户名再次提交时，会更新同一位面试官和它绑定的专属题库，不再每次生成一个新的 interviewer id

### 4. 数据库题库正式接管出题

- 面试题主链路已切换为数据库唯一出题源
- 运行时规则为：
  - 开场题走通用题库
  - 中间专业题优先走面试官专属题库
  - 专属题不足时回退通用题库
  - 收尾题走通用题库
- 新增正式题库表和使用记录表：
  - `formal_question_bank`
  - `formal_question_usage`

### 5. 后端八股题库更新

- 后端岗位默认通用题库已替换为新的 6 组 60 题：
  - Java / 语言基础
  - MySQL
  - Redis
  - 计算机网络与系统
  - 分布式与微服务
  - 算法与数据结构
- 默认 `secondme_tech` 也同步更新了 3 道示例专属题，用于验证“专属题优先、通用题兜底”

### 6. 反馈页信息增强

- 反馈页现在已经展示：
  - 总体总结
  - 改进建议
  - 每一轮问题
  - 候选人回答
  - 本轮评价
  - 参考答案

### 7. 稳定性修复

- 修复了 SecondMe 实时链路里 prompt 回显到题目内容的问题
- 修复了 prompt 过滤过宽导致的 `502 Bad Gateway` 超时问题
- 修复了管理台通用题库接口因旧进程未重启导致的 `404`

## 验证结果

- 前端构建通过：`npm run build`
- 后端测试通过：`cd apps/api && ./.venv/bin/python -m unittest discover -s tests`
- 当前本地 API 已验证健康检查正常：`http://127.0.0.1:8000/health`

## 当前风险和待补项

- 隐藏管理台仍无正式鉴权，不能直接暴露公网
- 豆包和 SecondMe Visitor Chat 仍依赖真实外部环境与测试凭证
- 反馈页还没有完整展示 `dimensions` 与 `suggestedAnswer`
- 运行时 session 仍是进程内状态，服务重启后无法续接
- 线上部署需要显式配置 `VITE_API_BASE_URL`

## 下周建议

- 完成豆包和 SecondMe Visitor Chat 的真实联调
- 为 `/admin/interviewers` 增加正式权限保护
- 补齐反馈页的维度评分和建议答案展示
- 继续整理题库运营方式，补批量导入和更清晰的分类编辑体验
