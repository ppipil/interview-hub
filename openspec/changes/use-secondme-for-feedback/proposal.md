## Why

The backend currently uses SecondMe to generate interview questions, but the final feedback is still produced locally through heuristic rules. This creates an inconsistent user experience and makes the round-by-round review feel disconnected from the actual avatar interviewer.

## What Changes

- Generate completed interview feedback through the configured SecondMe avatar instead of the local heuristic service
- Extend the feedback contract to include per-round review notes returned from the backend
- Update the frontend feedback page to render avatar-generated round reviews instead of deriving them locally

## Capabilities

### New Capabilities

- `secondme-feedback-generation`: Defines how completed interview feedback is requested from the configured SecondMe avatar and returned in structured form

### Modified Capabilities

- `secondme-interview-backend`: Completed-session feedback is generated by the active SecondMe avatar session, not by local heuristic rules

## Impact

- `apps/api/**`
- `apps/web/**`
- `docs/openapi.yaml`
- Feedback generation flow, parsing strategy, and feedback page data binding
