## ADDED Requirements

### Requirement: Completed interview feedback SHALL be generated through the configured SecondMe avatar

The backend SHALL use the active configured SecondMe avatar session to generate completed-session feedback rather than composing a local heuristic report.

#### Scenario: Frontend requests feedback for a completed session

- **GIVEN** an interview session has reached `completed`
- **AND** the backend still has the transcript and runtime context in memory
- **WHEN** the frontend calls `GET /api/v1/interview-sessions/{sessionId}/feedback`
- **THEN** the backend SHALL ask the configured SecondMe avatar to generate the report
- **AND** the backend SHALL parse and cache the resulting structured feedback before returning it

### Requirement: Feedback contract SHALL include round-by-round review data

The feedback payload SHALL include per-round review items generated by the avatar so the frontend can render a true round review instead of synthesizing local notes.

#### Scenario: Frontend renders a generated feedback report

- **WHEN** the backend returns a completed feedback payload
- **THEN** the payload SHALL contain `summary`, `dimensions`, `strengths`, `improvements`, `suggestedAnswer`, and `roundReviews`
- **AND** each `roundReviews` item SHALL include the round number, the question text, the user answer text, and a review note
