## Why

The PRD currently describes the intended MVP scope, but it does not indicate which items are already implemented, intentionally simplified, or still missing. As implementation has progressed, this creates ambiguity for planning and weekly execution because the document reads like everything is still pending.

## What Changes

- Add implementation status guidance to the PRD
- Mark major scope items and pages as completed, partially completed, intentionally adjusted, or pending
- Add a dated status snapshot so teammates can quickly align on what remains

## Capabilities

### Modified Capabilities

- `product-planning-docs`: The PRD communicates both target scope and current implementation status
