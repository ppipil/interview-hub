## MODIFIED Requirements

### Requirement: The PRD SHALL expose current implementation status alongside MVP scope

The PRD SHALL remain the source of MVP product intent while also showing which scoped capabilities are already implemented, partially implemented, intentionally simplified, or still pending.

#### Scenario: Team reads the PRD to plan the next work cycle

- **WHEN** a teammate reviews the PRD
- **THEN** they SHALL be able to distinguish product target scope from current implementation status
- **AND** the document SHALL show status markers for major scope items and pages
- **AND** the document SHALL indicate when the status snapshot was last updated
